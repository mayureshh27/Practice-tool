# Memory Events Start With Learning Loop Events

Shared Learning Memory will start by recording Source Ingested, Artifact Generated, Practice Attempted, Hint Requested, Blind Spot Detected, Concept Mastery Updated, and Session Summary Created events. User preferences and custom rules are expected to become important as the product evolves, but they do not block the first practice-centered version.

## Consequences

Memory events must be inspectable and tied to source, chunk, concept, artifact, and session identifiers where possible. Later custom engineering can refine scoring, compression, and preference handling without losing the historical learning trail.
